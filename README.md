## Sobre el Curso

Este curso de extensión está diseñado para cerrar la brecha entre la **programación académica** y la **ingeniería de software profesional**. A través de una metodología basada en proyectos reales, los estudiantes experimentarán simulaciones de entornos laborales, adoptando roles técnicos rotativos y aplicando flujos de trabajo colaborativos estándar de la industria.

### Filosofía del Curso

| **De:**                            | **Hacia:**                    |
| ---------------------------------- | ----------------------------- |
| Proyectos individuales             | Desarrollo colaborativo       |
| Archivos comprimidos por email     | Repositorios distribuidos     |
| `proyecto_final_v3_definitivo.zip` | Historial completo y trazable |
| Sin control de versiones           | Automatización de procesos    |
| Trabajo aislado                    | Equipos sincronizados         |

## Características del Curso

### Agnóstico del Lenguaje

El curso es **independiente del lenguaje de programación**. Aunque se sugiere inicialmente **Java + Maven** por facilidad de integración con workflows de CI/CD, los estudiantes pueden aplicar Git y CI/CD a cualquier stack tecnológico:

- Python + pytest
- JavaScript/TypeScript + npm
- C++ + CMake
- Go + módulos nativos
- Cualquier otro stack

**Compatibilidad:** Linux, Windows y macOS

### Metodología Práctica

<table>
<tr>
<td><strong>Simulación Laboral</strong><br/>Trabajo en células de 3 personas con roles rotativos semanales</td>
<td><strong>Roles Técnicos</strong><br/>Tech Lead (revisa y aprueba) + Developers (implementan features)</td>
<td><strong>Proyectos Reales</strong><br/>Desarrollo incremental con entregas parciales auditadas</td>
</tr>

## Estructura del Curso

El curso se organiza en **10 semanas** (40 horas aproximadamente) divididas en 4 módulos:

### Módulo 1: Fundamentos y Estandarización
**Semanas 1-3**

- Configuración del entorno de desarrollo
- Arquitectura de Git (Local, Staging, Remote)
- Ciclo de vida del código y commits semánticos
- Gestión del historial y versionamiento

### Módulo 2: Ramas y Estrategias de Colaboración
**Semanas 4-6**

- Branching y gestión de ramas
- Merging y resolución de conflictos
- Trabajo remoto y autenticación SSH
- Git Flow profesional

### Módulo 3: Calidad y Gestión de Proyectos
**Semanas 7-8**

- Code Review y Pull Requests
- Project Management (Issues, Labels, Milestones)
- Trazabilidad de cambios

### Módulo 4: Automatización e Integración Continua
**Semanas 9-10**

- Introducción a CI/CD y pipelines
- Automatización de pruebas
- Continuous Deployment y Releases

## Evaluación y Certificación

El curso **no contempla exámenes teóricos tradicionales**. La certificación se otorga bajo el modelo de **Validación de Competencias por Evidencia**.

### Criterios de Aprobación

| Criterio | Peso |
|----------|------|
| Historial de commits semánticos y atómicos | 25% |
| Pull Requests documentados (mínimo 5) | 20% |
| Pipeline CI/CD funcional | 20% |
| Proyecto compilable/ejecutable | 15% |
| Issues gestionados (mínimo 8) | 10% |
| Rotación de roles completada | 10% |

**Puntuación mínima:** 70/100 puntos

**Entregable:** Repositorio público auditado en GitHub/GitLab

## Recursos y Plataformas

Cada sesión combina:

- **15-20 minutos:** Exposición teórica breve

**Enfoque:** Todo se ejecuta en terminal (no GUI), fomentando la comprensión profunda de los comandos y flujos de trabajo.
